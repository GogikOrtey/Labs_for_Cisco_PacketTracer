Лабораторная работа 8.1
Цель

  • изучение работы протокола OSPF

Ход работы

Подсказка: чтобы окно описания лабораторной работы располагалось поверх всех окон поставьте галочку Top в левом нижнем углу.


Шаг 1. Назначение адресов.

  1. Назначте всем устройствам произвольные IP-адреса из сетей, указанных на схеме сети. Укажите верную маску и шлюз по умолчанию для всех ПК.
  2. Убедитесь в том, что интерфейсы маршрутизаторов включены (show ip interface brief).

Шаг 2. Настройка протокола OSPF.

  1. Включите протокол OSPF глобально (router ospf 1).
  2. Укажите интерфейсы, которые будут участвовать в работе протокола (network <адрес сети> <обратная маска> area 0).
  3. Укажите интерфейсы, которые не будут рассылать Hello-пакеты (passive-interface <интерфейс>).
  4. Выйдите из режима настройки протокола маршрутизации (end).

Шаг 3. Проверка работы протокола OSPF.

  1. Убедитесь, что OSPF включен на требуемых интерфейсах (show ip ospf interface).
  2. Убедитесь, что маршрутизатор обнаружил все соседние маршрутизаторы и находится со всеми из них в состоянии Full (show ip ospf neighbor).
  3. Убедитесь, что в таблице маршрутизации присутствуют маршруты ко всем сетям (show ip route).
  4. Разорвите любое из соединений между маршрутизаторами и обратите внимание, как изменится при этом таблица маршрутизации.

Шаг 4. Изменение метрики маршрутов.

  1. Измените стоимость каналов таким образом, чтобы пакеты между сетями 192.168.2.0 и 192.168.1.0 проходили через верхний маршрутизатор всегда, когда это возможно, а в случае отказа одного из каналов Постышева-Уссурийск или Луговая-Уссурийск - проходили по каналу Постышева-Луговая. Для этого воспользуйтесь командой ip ospf cost <стоимость> в режиме конфигурирования интерфейса. Проверить текущую стоимость каналов можно командой show ip ospf interface.
  2. Проверьте правильность маршрутизации утилитой tracert или в режиме симуляции.

Полезные команды:

show ip ospf database - показать базу данных состояния каналов (LSDB)

debug ip ospf events - включение подробного протоколирования всех действий OSPF

undebug all - выключение подробного протоколирования
