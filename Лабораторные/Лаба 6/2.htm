Лабораторная работа 6.2
Цель

  • настройка маршрутизации между VLAN с использованием маршрутизатора

Ход работы

Подсказка: чтобы окно описания лабораторной работы располагалось поверх всех окон поставьте галочку Top в левом нижнем углу.


Шаг 1. Настройка sub-интерфейсов.

  1. Настройка маршутизации между VLAN осуществляется с помощью виртуальных sub-интерфейсов. Настройка таких интерфейсов аналогична настройке физических интерфейсов маршрутизатора. Единственное отличие заключается в необходимости указания типа инкапсуляции - dot1q и номера VLAN.
  2. На маршрутизаторе Router0 введите следующие команды:

    Router#conf t
    Router(config)#int Fa 0/0
    Router(config-if)#no shutdown
    Router(config-if)#exit

    Router(config)#int Fa 0/0.10
    Router(config-subif)#encapsulation dot1Q 10
    Router(config-subif)#ip address 192.168.10.254 255.255.255.0
    Router(config-subif)#exit

    Router(config)#int Fa 0/0.20
    Router(config-subif)#encapsulation dot1Q 20
    Router(config-subif)#ip address 192.168.20.254 255.255.255.0
    Router(config-subif)#exit

    Router(config)#int Fa 0/0.30
    Router(config-subif)#encapsulation dot1Q 30
    Router(config-subif)#ip address 192.168.30.254 255.255.255.0
    Router(config-subif)#end
    Router# copy run start
    Router#

  3. Проверьте правильность настройки интерфейсов, воспользовавшись командой show ip interface brief (или сокращенно sh ip int br).
  4. Переведите интерфейс коммутатора, к которому подключается маршрутизатор, в режим транка.

Шаг 2. Настройка рабочих станций и сервера.

  1. Настройте на рабочих станциях и сервере адрес шлюза - адрес соответствующего sub-интерфейса маршрутизатора.

Шаг 3. Проверка соединения.

  1. Убедитесь в том, что команда ping с одного из компьютеров до всех остальных компьютеров выполняется успешно. Если это не так, проверьте правильность настройки sub-интерфейсов маршрутизатора и адрес шлюза на соответствующем компьютере..
  2. Обратите внимание, как выглядит таблица маршрутизации на маршрутизаторе (show ip route).
  3. Перейдите в режим симуляции. В списке фильтров оставьте только ICMP. Отправьте пакет с одного из компьютеров на любой, находящийся в другом VLAN. Все пакеты, отправляемые в другие VLAN обязательно проходят через маршрутизатор.

Шаг 4. Проверка результатов.

Нажмите кнопку Check Results чтобы проверить правильность выполнения работы.
