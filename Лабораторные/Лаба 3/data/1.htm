Цель

Ход работы

Подсказка: чтобы окно описания лабораторной работы располагалось поверх всех окон поставьте галочку Top в левом нижнем углу.


Шаг 1. Построение сети.

  1. Создайте сеть, состоящую из моста, двух хабов и четырех компьютеров, как показано на рисунке:
  2. Настройте рабочие станции согласно таблице:
    Станция IP-адрес Маска подсети
    PC0 172.16.0.1 255.255.255.0
    PC1 172.16.0.2 255.255.255.0
    PC2 172.16.0.3 255.255.255.0
    PC3 172.16.0.4 255.255.255.0

    Для этого выберите компьютер, перейдите на вкладку Desktop, щелкните IP configuration и введите необходимые значения..

Шаг 2. Проверка соединения.

  1. Убедитесь в том, что команда ping с PC0 до других компьютеров выполняется успешно. Если это не так, проверьте правильность настройки IP-адреса и маски подсети.
  2. Определите MAC адрес каждого компьютера.
    1. В командной строке (Command Prompt) введите команду ipconfg /all.
    2. Сохраните значение поля Physical Address, воспользовавшись инструментом Place Note ().

Шаг 3. Анализ процесса обучения моста.

  1. Отобразите таблицу MAC адресов моста:
    1. Выберите инструмент Inspect на панели инструментов ().
    2. Щелкните по мосту и в появившемся меню выберите MAC Table. Откроется таблица коммутации моста.
    3. Если в таблице присутствуют записи, подождите, пока они пропадут. Время жизни записей в таблице коммутации составляет 300 секунд. Чтобы таблица очистилась быстрее, можно несколько раз нажать кнопку Fast Forward Time, расположенную над панелью выбора устройств.
  2. Перейдите в режим Simulation.
  3. В поле Visible Events (отображаемые события) нужно оставить только события ICMP ( этот протокол используется утилитой ping). Для этого нажмите кнопку Edit Filters, снимите галочку с Show All/None и поставьте галочку в поле ICMP. Щелкните в любом месте за пределами окна выбора событий чтобы закрыть окно.
  4. Выберите инструмент Add Simple PDU () и щелкните последовательно компьютеры PC0 и PC2.
  5. Нажмите кнопку Auto Capture / Play чтобы запустить симуляцию. Обратите внимание, как мост запоминает MAC адреса отправителя при прохождении через него фрейма. Во время симуляции можно посмотреть структуру пересылаемого фрейма. Для этого в главном окне щелкните по пересылаемому фрейму. В открывшемся окне во вкладках Inbound PDU Details и Outbound PDU Details отображается структура PDU (Protocol Data Unit) всех задействованных в пересылке уровней. В данном случае можно увидеть Ethernet-фрейм, IP-пакет и ICMP-пакет.

Шаг 4. Анализ процесса фильтрации моста.

  1. Нажмите кнопку Delete на панели симуляции, чтобы удалить текущий сценарий.
  2. В командной строке компьютера PC0 введите команду ping 172.16.0.2 и нажмите клавишу Enter.
  3. Вернитесь в окно симуляции.
  4. Нажмите кнопку Auto Capture / Play чтобы запустить симуляцию. Таблица коммутации моста пуста, поэтому фрейм будет отправлен в другой сегмент, а адрес источника (PC0) будет помещен в таблицу коммутации. Затем последует ответный фрейм от PC1. Адрес PC1 будет помещен в таблицу коммутции. Адрес получателя мост уже знает - он находится в том же самом порту, что и источник. Значит фрейм нужно отфильтровать, то есть отбросить. Когда PC0 отправит второй фрейм компьютеру PC1, мост этот фрейм отфильтрует, поскольку и отправитель и получатель находятся за одним и тем же портом.

Шаг 5. Проверка результатов.

Нажмите кнопку Check Results чтобы проверить правильность выполнения работы.
