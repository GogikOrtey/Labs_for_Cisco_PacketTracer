Лабораторная работа 4.1
Цель

  • построение беспроводной сети 802.11
  • изучение коммутации в смешанной Ethernet - 802.11 сети

Ход работы

Подсказка: чтобы окно описания лабораторной работы располагалось поверх всех окон поставьте галочку Top в левом нижнем углу.


Шаг 1. Построение сети.

  1. Создайте сеть, состоящую из трех коммутаторов, двух точек доступа (Access Point-PT), двух ноутбуков, КПК (PDA-PT), планшетного ПК (TabletPC-PT), компьютера и сервера, как показано на рисунке:
  2. Настройте рабочие станции согласно таблице:
    Станция IP-адрес Маска подсети
    Server0 192.168.100.1 255.255.255.0
    PC0 192.168.100.2 255.255.255.0
    Pda0 192.168.100.3 255.255.255.0
    Tablet PC0 192.168.100.4 255.255.255.0
    Laptop0 192.168.100.5 255.255.255.0
    Laptop1 192.168.100.6 255.255.255.0

    Для этого выберите компьютер, перейдите на вкладку Desktop, щелкните IP configuration и введите необходимые значения..

Шаг 2. Проверка соединения.

  1. Убедитесь в том, что команда ping с компьютера PC0 до сервера Server0 выполняется успешно. Если это не так, проверьте правильность настройки IP-адреса и маски подсети.
  2. Определите MAC адрес каждого компьютера.
    1. В командной строке (Command Prompt) введите команду ipconfg /all.
    2. Сохраните значение поля Physical Address, воспользовавшись инструментом Place Note ().

Шаг 3. Подготовка компьютеров к работе в беспроводной сети.

По умолчанию ноутбуки не имеют встроенного адаптера Wi-Fi, поэтому, чтобы они могли работать в созданной сети, необходимо эти адаптеры подключить вручную:

  1. Выберите ноутбук и перейдите на вкладку Physical.
  2. Отключите питание ноутбука, нажав кнопку питания.
  3. Перетащите Ethernet-адаптер ноутбука на изображение модуля в правом нижнем углу.
  4. В списке модулей выберите PT-LAPTOP-NM-1W.
  5. Перетащите название модуля из списка, либо его изображение в освободившийся слот ноутбука.
  6. Включите питание ноутбука, нажав кнопку питания.

Шаг 4. Настройка параметров беспроводной сети.

  1. Создание BSS на основе точки доступа Access Point0:
    1. Выберите точку доступа Access Point0.
    2. На вкладке Config выберите Port1.
    3. Задайте SSID: Network1.
    4. Аутентификация WPA2-PSK. Парольная фраза: Network1.
    5. Тип шифрования: AES.
  2. Укажите эти же параметры на Pda0 и Tablet PC0. Если параметры указаны верно, эти устройства должны подключиться к точке доступа Access Point0.
  3. Подобным образом настройте точку доступа Access Point1 и два ноутбука. При этом укажите SSID и парольную фразу: Network2. Если параметры указаны верно, оба ноутбука должны подключиться к точке доступа Access Point1.
  4. Убедитесь в том, что команда ping с каждого беспроводного устройства до сервера Server0 выполняется успешно. Если это не так, проверьте правильность настройки IP-адреса, маски подсети, а также параметров беспроводной сети.

Шаг 5. Анализ процесса коммутации в беспроводной сети.

  1. Перейдите в режим Simulation.
  2. В поле Visible Events (отображаемые события) нужно оставить только события ICMP.
  3. Выберите инструмент Add Simple PDU () и щелкните последовательно Pda0 и Tablet PC0.
  4. Нажмите кнопку Auto Capture / Play чтобы запустить симуляцию. Обратите внимание, что точка доступа, подобно хабу, рассылает полученный фрейм всем подключенным к ней устройствам.
  5. Нажмите кнопку Delete на панели симуляции, чтобы удалить текущий сценарий.
  6. Выполните пункты 3 - 5 для пар Pda0 - Server0 и Pda0 - Laptop1. Проанализируйте процесс коммутации в смешанной сети.

Шаг 5. Проверка результатов.

Нажмите кнопку Check Results чтобы проверить правильность выполнения работы.
