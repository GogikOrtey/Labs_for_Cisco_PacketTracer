Лабораторная работа 5.3
Цель

  • Разработка и внедрение IP-адресации
  • Настройка статических маршрутов

Ход работы

Подсказка: чтобы окно описания лабораторной работы располагалось поверх всех окон поставьте галочку Top в левом нижнем углу.


Шаг 1. Разработка IP-адресации.

  1. Для разработки адресации сети предприятия вам была выделена сеть 192.168.0.0/24. Полученную сеть необходимо разделить на подсети таким образом, чтобы образовалось как минимум 4 подсети по 30 IP-адресов в каждой.
  2. Последнюю из полученных подсетей необходимо в свою очередь разделить как минимум на 3 подсети по 2 IP-адреса в каждой.
  3. В случае правильного разбиения должно получиться 15 подсетей.

Шаг 2. Внедрение IP-адресации.

  1. Первый доступный адрес из первой крупной подсети назначте интерфейсу Fa0/0 маршрутизатора Router1. Последний доступный адрес назначте компьютеру PC1.
  2. Первый доступный адрес из второй крупной подсети назначте интерфейсу Fa0/0 маршрутизатора Router2. Последний доступный адрес назначте компьютеру PC0.
  3. Первый доступный адрес из третьей крупной подсети назначте интерфейсу Fa0/0 маршрутизатора Router3. Последний доступный адрес назначте компьютеру PC2.
  4. Первый доступный адрес из четвертой крупной подсети назначте интерфейсу Fa0/1 маршрутизатора Router3. Последний доступный адрес назначте компьютеру PC3.
  5. На всех компьютерах настройте верный шлюз по умолчанию.
  6. Результат команды ping между компьютером и соответствующим шлюзом по умолчанию должен быть успешным.
  7. Первый доступный адрес из первой малой подсети назначте интерфейсу Se0/0 маршрутизатора Router2. Второй адрес назначте интерфейсу Se0/0 маршрутизатора Router1.
  8. Первый доступный адрес из второй малой подсети назначте интерфейсу Se0/1 маршрутизатора Router1. Второй адрес назначте интерфейсу Se0/0 маршрутизатора Router3.
  9. Первый доступный адрес из третьей малой подсети назначте интерфейсу Se0/1 маршрутизатора Router2. Второй адрес назначте интерфейсу Se0/1 маршрутизатора Router3.

Шаг 3. Настройка статических маршрутов.

Настройте на каждом из маршрутизаторов статические маршруты ко всем подсетям, к которым они не подключены напрямую.

Шаг 4. Проверка соединения.

Проверьте наличие соединения между всеми компьютерами с помощью команды ping.

Шаг 5. Проверка результатов.

Нажмите кнопку Check Results чтобы проверить правильность выполнения работы.
